# 使用说明

## 一、首次安装

1. 保持 AstrBot 和 QQ 官方机器人在线。
2. 双击 `启动助手.bat`。首次启动会生成 `config/config.json` 和随机桥接令牌。
3. 双击 `安装AstrBot插件.bat`；随后在 AstrBot WebUI 的插件页重载“BOSS 求职助手”，或重启 AstrBot。
4. 打开 AstrBot WebUI → 设置 → API Keys，创建只包含 `im` 权限的 Key。复制后写入 `config/config.json` 的 `astrbot.apiKey`，然后重启本地助手。
5. 在 QQ 中向机器人发送 `/求职助手 绑定`，把当前私聊或群聊设为通知目标。
6. 在 Tampermonkey 中安装 `dist/boss-qq-assistant.user.js`。
7. 打开 BOSS 直聘岗位列表，在侧边面板“QQ 通知桥接”中填写 `config/config.json` 的 `bridge.token`，点击“测试连接”。

## 二、日常使用

1. 双击 `启动助手.bat`，确认 AstrBot/QQ 在线。
2. 登录 BOSS 直聘并进入岗位搜索列表。
3. 配置岗位、公司、Boss 活跃度、问候语、间隔、重试与最大沟通数量。
4. 点击“开始任务”。遇到验证码、登录失效或达到每日上限时会自动停止；不要尝试绕过验证。

## 三、QQ 命令

- `/求职助手 绑定`
- `/求职助手 状态`
- `/求职助手 暂停`
- `/求职助手 开始`
- `/求职助手 今日记录`
- `/求职助手 上限 20`
- `/求职助手 帮助`

QQ 的“开始/暂停”控制本地许可状态；真正开始自动浏览仍需在 BOSS 网页点击“开始任务”。

## 四、数据与导出

网页脚本继续使用 IndexedDB 保存岗位记录，并支持 JSON/Excel 导出。桥接服务将通知事件保存在 `bridge/data/records.json`，可通过 `GET /api/v1/export.csv`（Bearer 桥接令牌）导出 CSV。删除 `bridge/data/records.json` 可清空桥接事件，网页岗位数据需从脚本面板清理。
